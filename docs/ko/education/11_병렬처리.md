# 병렬 처리와 비동기
> `parallel for`로 컬렉션을 병렬 처리하고, `simd for`로 수치 루프를 벡터화하고, `async/await`와 `spawn`으로 비동기 작업을 처리한다.

---

## 루프 접두사 체계

dri는 `for` 앞에 접두사를 붙여 실행 방식을 선언합니다.

```
for (...)             { }   # 일반 순차 실행
parallel for (...)    { }   # Work-Stealing 병렬 실행
simd for (...)        { }   # CPU SIMD 벡터화
parallel simd for (...) { } # 병렬 + 벡터화 동시 적용
```

## 병렬 처리 선택 가이드

| 상황 | 방법 |
|------|------|
| 컬렉션 원소를 독립 처리 | `parallel for (x of ...)` |
| 수치 계산 집약 루프 | `simd for (i in 0..N)` |
| 수치 계산 + 병렬 동시 | `parallel simd for (i in 0..N)` |
| 백그라운드 I/O | `async`/`await` |
| 결과 불필요 백그라운드 태스크 | `spawn { }` |

## parallel for — 병렬 반복

```
parallel for (변수 of 컬렉션) { 본문 };
parallel for (변수 in 시작..끝) { 본문 };
```

Work-stealing 스레드 풀 기반으로 각 반복을 병렬 실행합니다.

```dri
int[] data = [1, 2, 3, 4, 5, 6, 7, 8];

parallel for (item of data) {
    print("처리:", item);
};
```

대규모 병렬 처리:

```dri
list<String> files = [];
lst.push(files, "a.txt");
lst.push(files, "b.txt");
lst.push(files, "c.txt");
lst.push(files, "d.txt");

parallel for (file of files) {
    String content = io.read_file(file);
    print("읽음:", file, "크기:", length(content));
};
```

## simd for — SIMD 벡터화

```
simd for (변수 in 시작..끝) { 본문 };
simd for (변수 of 컬렉션) { 본문 };
```

CPU의 AVX2/AVX-512(x86) 또는 NEON(ARM)을 자동 활용합니다.

```dri
int N = 1024;
double[] a = new double[N];
double[] b = new double[N];
double[] c = new double[N];

simd for (i in 0..N) {
    c[i] = a[i] + b[i];
};
```

`@noalias`와 함께 사용하면 더 공격적인 벡터화가 가능합니다:

```dri
@noalias
void saxpy(double alpha, double[] x, double[] y, int n) {
    simd for (i in 0..n) {
        y[i] = alpha * x[i] + y[i];
    };
};
```

## parallel for reduction — 병렬 리덕션

병렬 루프에서 여러 스레드가 하나의 변수에 누적 연산을 하면 **데이터 경쟁(Race Condition)**이 발생합니다.  
`reduction` 키워드는 각 스레드별 지역 변수로 연산한 뒤 루프 종료 시 **원자적으로 병합**합니다.

```
parallel for (변수 of 컬렉션) reduction(연산자:변수) { 본문 };
```

| 연산자 | 초기값 | 설명 |
|--------|--------|------|
| `+` | `0` | 합산 |
| `*` | `1` | 곱셈 |
| `max` | 최솟값 | 최댓값 |
| `min` | 최댓값 | 최솟값 |
| `and` | `true` | 논리 AND |
| `or` | `false` | 논리 OR |

```dri
int[] big_array = new int[1000000];
for (i in 0..1000000) { big_array[i] = i; };

mut int total = 0;
parallel for (item of big_array) reduction(+:total) {
    total += item;
};
print("합계:", total);

# 최댓값 리덕션
mut int max_val = 0;
parallel for (item of big_array) reduction(max:max_val) {
    if (item > max_val) { max_val = item; };
};
print("최댓값:", max_val);
```

여러 리덕션 동시 사용:

```dri
mut double sum = 0.0;
mut double sum_sq = 0.0;

parallel for (x of data) reduction(+:sum) reduction(+:sum_sq) {
    sum += x;
    sum_sq += x * x;
};

double mean = sum / lst.size(data) as double;
double variance = sum_sq / lst.size(data) as double - mean * mean;
```

> C++ OpenMP의 `reduction(+:sum)` 패턴과 동일한 의미론입니다.

## parallel simd for — 병렬 + 벡터화 조합

병렬 실행과 SIMD 벡터화를 동시에 적용합니다. 대규모 수치 계산의 핵심 패턴입니다.

```dri
int N = 1048576;   # 1M 원소
double[] a = new double[N];
double[] b = new double[N];
double[] result = new double[N];

@noalias
parallel simd for (i in 0..N) {
    result[i] = a[i] * 2.0 + b[i];
};
```

스레드당 청크 크기를 SIMD 너비에 맞게 자동 정렬합니다.

## async / await

`async` 함수 내에서 `await`로 I/O를 논블로킹 처리합니다.

```dri
async void fetch(String path) {
    String data = await io.read_file(path);
    print("데이터:", length(data), "바이트");
};

fetch("data.txt");
```

여러 비동기 작업 순차 실행:

```dri
async void load_all() {
    String config = await io.read_file("config.json");
    String data   = await io.read_file("data.csv");
    print("설정:", length(config), "바이트");
    print("데이터:", length(data), "바이트");
};

load_all();
```

async + try/catch:

```dri
async void safe_fetch(String path) {
    try {
        String content = await io.read_file(path);
        print("성공:", length(content), "바이트");
    } catch (err) {
        print("읽기 실패:", err);
    };
};

safe_fetch("existing.txt");
safe_fetch("missing.txt");
```

## spawn — 파이어 앤 포겟

```
spawn { 본문 };
```

결과를 기다리지 않고 백그라운드에서 실행합니다.

```dri
spawn {
    print("백그라운드 시작");
    for (i in 0..1000000) {
        # 무거운 계산
    };
    print("백그라운드 완료");
};

print("메인 계속 실행");
```

여러 spawn 동시 시작:

```dri
spawn {
    String log = io.read_file("events.log");
    process_log(log);
};

spawn {
    String cfg = io.read_file("config.json");
    reload_config(cfg);
};

spawn {
    sys.exec("backup.sh");
};
```

## CPU 친화성

특정 코어에 스레드를 고정해 NUMA/캐시 효율을 높입니다.

```dri
sys.affinity(0);    # 코어 0에 고정

simd for (i in 0..1000000) {
    result[i] = compute(i);
};
```

## 전체 예시 — 병렬 행렬 곱셈

```dri
int N = 256;
double[] A = new double[N * N];
double[] B = new double[N * N];
double[] C = new double[N * N];

# 초기화
for (i in 0..N) {
    for (j in 0..N) {
        A[i * N + j] = i as double + j as double;
        B[i * N + j] = i as double - j as double;
    };
};

double t0 = perf.now();

# 행 단위 parallel for
int[] rows = new int[N];
for (i in 0..N) {
    rows[i] = i;
};

parallel for (row of rows) {
    int i = row;
    for (j in 0..N) {
        double s = 0.0;
        simd for (k in 0..N) {
            s = simd.fmadd(A[i * N + k], B[k * N + j], s);
        };
        C[i * N + j] = s;
    };
};

double t1 = perf.now();
print("행렬 곱셈 완료:", t1 - t0, "ms");
```

## 스레드 안전성 분석기

dri 컴파일러는 빌드 시 **정적 스레드 안전성 분석**을 자동 실행합니다.

### 감지하는 패턴

**1. 동기화 없는 공유 변수 쓰기**

```dri
int total = 0;

parallel for (x of data) {
    total += x;   # 경고: total에 atomic/reduction 없이 쓰기
};
```

컴파일러 경고:
```
warning: possible thread-unsafe write to 'total' inside parallel for
         — add reduction(+:total) or atomic<int>, or use sys.sync.fence_full()
```

**해결 방법 1: reduction**
```dri
int total = 0;
parallel for reduction(+:total) (x of data) {
    total += x;   # OK — OpenMP reduction으로 변환됨
};
```

**해결 방법 2: atomic**
```dri
atomic<int> total = 0;
parallel for (x of data) {
    sys.sync.atomic_store(total, sys.sync.atomic_load(total) + x);
};
```

### 분석기 비활성화

```bash
dri main.dri --exe app --no-analyze   # 정적 분석 건너뜀 (릴리스 빌드 속도 향상)
```

## 하드웨어 토폴로지 인식 스케줄러 (sys.topology)

현대 CPU는 동일한 L3 캐시를 공유하는 코어들이 다이(Die) 또는 CCD(Core Complex Die) 단위로 묶여 있습니다. `sys.topology.*`를 사용하면 이 구조를 인식하여 스레드를 최적 위치에 배치할 수 있습니다.

### Intel P-core / E-core 인식

```dri
use sys;

list<int> fast_cores = sys.topology.p_cores();  # 고성능 코어 ID 목록
list<int> eff_cores  = sys.topology.e_cores();  # 효율 코어 ID 목록
int total = sys.topology.total();
String vendor = sys.topology.vendor();  # "Intel" / "AMD" / "other"

print("총 논리 코어:", total);
print("P-코어:", fast_cores);
print("E-코어:", eff_cores);
```

### AMD CCD(L3 캐시 공유 그룹)

```dri
use sys;

# L3 캐시를 공유하는 코어 그룹 목록 (AMD CCD, Intel Ring 등)
list<list<int>> groups = sys.topology.cache_groups();

for (i in 0..length(groups)) {
    print("캐시 그룹", i, ":", groups[i]);
};
```

### 병렬 처리 최적화 패턴

```dri
use sys;

# parallel for 실행 전 L3-공유 코어에 스레드 밀집
sys.topology.pin_threads();   # 자동으로 최적 캐시 그룹에 고정

int[] data = range(1000000);
int total = 0;

parallel for reduction(+:total) (x of data) {
    total += x * x;
};

print("결과:", total);
```

### sys.topology API

| 함수 | 반환형 | 설명 |
|------|--------|------|
| `sys.topology.p_cores()` | `list<int>` | 고성능(P) 코어 ID 목록 |
| `sys.topology.e_cores()` | `list<int>` | 효율(E) 코어 ID 목록 |
| `sys.topology.cache_groups()` | `list<list<int>>` | L3 캐시 공유 그룹 |
| `sys.topology.total()` | `int` | 전체 논리 코어 수 |
| `sys.topology.vendor()` | `String` | CPU 제조사 문자열 |
| `sys.topology.pin_threads()` | `void` | 현재 스레드를 최적 그룹에 고정 |
| `sys.topology.set_affinity(n)` | `void` | 특정 코어 n에 현재 스레드 고정 |

### 효과

- **Intel 12세대+**: P-코어만 병렬 루프에 배정 → E-코어의 느린 빈도 회피
- **AMD Ryzen (다중 CCD)**: 동일 CCD 코어에 스레드 집중 → 크로스-CCD 레이턴시(~80ns) 제거
- **일반**: L3 공유 그룹 내에서만 스레드 배치 → 캐시 미스 급감
