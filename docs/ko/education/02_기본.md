### 타입 추론 - ```var```
```dri

```

### `var` 허용 범위

| 위치 | `var` 허용 여부 | 이유 |
|------|----------------|------|
| 지역 변수 | ✅ | 우변 표현식으로 타입 확정 가능 |
| for 루프 변수 | ✅ | 컬렉션 원소 타입으로 확정 가능 |
| **함수 반환 타입** | ❌ | 컴파일러가 호출 측 타입 검사 불가 |
| **함수 매개변수** | ❌ | 오버로딩·제네릭 해석 불가, 트랜스파일 비용 급증 |
| **클래스 필드** | ❌ | 구조체 레이아웃 컴파일 타임 확정 불가 |

```dri
# 허용
var result = compute();           # 지역 변수
for (var item of list) { ... };   # 루프 변수

# 컴파일 오류
var add(var a, var b) { ... };    # 오류: 매개변수·반환 타입에 var 불가

class Foo {
    var value;                     # 오류: 필드에 var 불가
};
```

> HPC 언어 특성상 함수 시그니처는 **반드시 타입을 명시**해야 합니다.  
> 이로써 C++ 트랜스파일러가 ABI와 오버로딩을 명확히 결정할 수 있습니다.

### 참조 타입
### `Ref<T>` — 공유 소유권 (`shared_ptr`)
```dri
Ref<String> s = new String();
Ref<MyClass> obj = new MyClass();
Ref<int[]> buf = new int[100];

# null 허용
Ref<String> maybe = null;
if (maybe != null) {
    print(maybe);
};
```

### `Own<T>` — 단일 소유권 (`unique_ptr`)

```dri
Own<int[]> buf = new int[1024];
Own<int[]> other = move buf;  # 소유권 이전 후 buf는 사용 불가
```

### `Borrow<T>` — 비소유 참조 (`const T&`)

```dri
void read_only(Borrow<int[]> data) {
    print(data[0]);
};
```

## `Option<T>` — 있거나 없거나

```dri
Option<int> safe_div(int a, int b) {
    if (b == 0) {
        return None;
    };
    return Some(a / b);
};

match safe_div(10, 2) {
    Some(v) => { print("결과:", v); };
    None    => { print("나눗셈 실패"); };
};
```

## `Result<T>` — 성공 또는 실패

```dri
Result<int> parse_num(String s) {
    if (length(s) == 0) {
        return Err("빈 문자열");
    };
    return Ok(str.to_int(s));
};

match parse_num("42") {
    Ok(v)  => { print("파싱 성공:", v); };
    Err(e) => { print("오류:", e); };
};
```

## `dim` — 물리 단위 타입


### 단위 연산 규칙

단위 간 사칙연산의 결과 타입은 아래 규칙을 따릅니다.

| 연산 | 조건 | 결과 타입 |
|------|------|----------|
| `A + B`, `A - B` | 같은 단위 | 동일 단위 |
| `A + B`, `A - B` | 다른 단위 | **컴파일 오류** |
| `A * B` | 임의 단위 | 복합 단위 (새 `dim` 정의 필요) |
| `A / B` | 임의 단위 | 복합 단위 (새 `dim` 정의 필요) 또는 `double` |
| `A * scalar` | — | 동일 단위 |
| `A / scalar` | — | 동일 단위 |

```dri
dim Meter = "m";
dim Second = "s";
dim MeterPerSecond = "m/s";   # 복합 단위 사전 정의

Meter dist = 10.0;
Second time = 2.0;

# 같은 단위 덧셈 — OK
Meter total = dist + Meter(3.0);   # 13.0 m

# 다른 단위 덧셈 — 컴파일 오류
# Meter wrong = dist + time;       # 오류: Meter + Second 불가

# 나누기 — 복합 단위로 수신하거나 double로 수신
MeterPerSecond speed = dist / time;   # 5.0 m/s (복합 단위 정의 있을 때)
double ratio = dist / time;           # 5.0 (단위 버림, double로 수신 시 허용)

# 스칼라 곱셈/나눗셈
Meter half = dist * 0.5;    # 5.0 m
Meter scaled = dist / 2.0;  # 5.0 m
```

미정의 복합 단위 결과:

```dri
# 복합 단위(dim)를 정의하지 않고 나눈 결과 → double로만 수신 가능
dim Kg = "kg";
Meter dist2 = 3.0;
Kg mass = 5.0;

double ratio2 = dist2 / mass;   # OK: 단위 버리고 double 수신
# Meter kg_ratio = dist2 / mass;  # 오류: 정의되지 않은 복합 단위
```

## 입출력
### 출력
```dri
print("Hello World!");          # >>> Hello World!
print(42);                      # >>> 42
print(true);                    # >>> true

int x = 10;                     # >>> x = 10 정의
String name = "dri";            # >>> name = "dri" 정의
print(name, " version: ", x);   # >>> dri version: 10
print(name + " version: " + x); # >>> dri version: 10
print(`{name} version: {x}`);   # >>> dri version: 10
print(name * 3);                # >>> dridridri
```

### 입력
```dri
String usrInput; # 왼쪽처럼 정의 시 null이 기본이다.
input("값을 입력하시오 >>> ", usrInput);
usrInput = input("값을 입력하시오 >>> ");
```
> `<변수> = input(...);`을 할 때,  
> `input(<String>, <args>);`의 형태입니다.
