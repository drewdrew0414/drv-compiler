# drv 언어 소개
### Learn like Python. Scale like C++.

---

## 언어 특성

drv는 다음과 같은 목표에 중점을 둔 언어입니다.

| 목표 | 비교 언어 | 설명 |
|------|---------|------|
| 높은 생산성 | Python | 직관적이고 간결한 문법 |
| 강한 안전성 | Rust | 소유권·대여 모델 + 타입 안전 |
| 최고 성능 | C++ | 제로 코스트 추상화, C++17 트랜스파일 |
| 수치 계산 | Julia | 내장 SIMD, tensor 연산, 자동 미분 |

---

## 빠른 시작 — Hello World

```drv
# hello.drv
print("Hello, drv!");

int x = 10;
double pi = 3.14159;
print("x =", x, "pi =", pi);

for (i in 0..5) {
    print(i);
};
```

컴파일 및 실행:

```bash
drv hello.drv --exe hello
./hello
```

출력:
```
Hello, drv!
x = 10 pi = 3.14159
0
1
2
3
4
```

## 작동 방식

drv 소스 코드는 C++17 코드로 **트랜스파일**된 후 C++ 컴파일러로 빌드됩니다.

```
hello.drv  →  [drv 컴파일러]  →  hello.cpp  →  [clang/g++]  →  hello
```

- SIMD·병렬 기능은 clang/g++ 의 최적화 파이프라인을 그대로 활용합니다.
- `#line` 지시문으로 오류 위치가 원래 `.drv` 파일을 가리킵니다.

## CLI 옵션

```
drv <입력.drv> [옵션]
```

| 옵션 | 설명 |
|------|------|
| `--exe <출력파일>` | 실행 파일 생성 |
| `--cpp <출력.cpp>` | C++ 트랜스파일 결과만 저장 |
| `--check` | 컴파일 없이 문법·타입 검사만 수행 |
| `--debug` | 디버그 빌드 (`-O0`, 범위 검사 활성화) |
| `--release` | 릴리스 빌드 (`-O3`, `-march=native`, LTO) |
| `--opt <0~3>` | 최적화 수준 수동 지정 |
| `--native` | `-march=native` 활성화 |
| `--lto` | Link-Time Optimization 활성화 |
| `--trace <파일>` | parallel 루프 추적 파일 생성 (Chrome DevTools 형식) |
| `-D<플래그>` | 컴파일 타임 플래그 정의 (static_if 조건에 사용) |

```bash
drv main.drv --exe main --release
drv main.drv --cpp out.cpp --check
drv main.drv --exe main --debug
drv main.drv --exe main --opt 2 --native --lto
drv main.drv --exe main --release --trace trace.json
```

## 언어 규칙 요약

| 규칙 | 내용 |
|------|------|
| 문장 종결자 | 모든 문장은 `;`으로 끝납니다 |
| 블록 | `{ }` 중괄호 필수 |
| 주석 | `#` 한 줄, `!# ... ##` 여러 줄 |
| 파일 확장자 | `.drv` |
| 진입점 | 파일 최상위에 작성 (별도 `main` 함수 불필요) |

```drv
!# 여러 줄 주석
   이렇게 작성합니다.
##

# 한 줄 주석
int x = 42;   # 인라인 주석
print(x);
```

> 변수·타입·입출력 기본 문법은 [[02_기본.md](02_기본.md)]를 참고하세요.
