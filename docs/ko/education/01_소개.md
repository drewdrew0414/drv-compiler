# dri 언어 소개
### Learn like Python. Scale like C++.

---

## 언어 특성

dri는 다음과 같은 목표에 중점을 둔 언어입니다.

| 목표 | 비교 언어 | 설명 |
|------|---------|------|
| 높은 생산성 | Python | 직관적이고 간결한 문법 |
| 강한 안전성 | Rust | 소유권·대여 모델 + 타입 안전 |
| 최고 성능 | C++ | 제로 코스트 추상화, C++17 트랜스파일 |
| 수치 계산 | Julia | 내장 SIMD, tensor 연산, 자동 미분 |

---

## 빠른 시작 — Hello World

```dri
# hello.dri
print("Hello, dri!");

int x = 10;
double pi = 3.14159;
print("x =", x, "pi =", pi);

for (i in 0..5) {
    print(i);
};
```

컴파일 및 실행:

```bash
dri hello.dri --exe hello
./hello
```

출력:
```
Hello, dri!
x = 10 pi = 3.14159
0
1
2
3
4
```

## 작동 방식

dri 소스 코드는 C++17 코드로 **트랜스파일**된 후 C++ 컴파일러로 빌드됩니다.

```
hello.dri  →  [dri 컴파일러]  →  hello.cpp  →  [clang/g++]  →  hello
```

- SIMD·병렬 기능은 clang/g++ 의 최적화 파이프라인을 그대로 활용합니다.
- `#line` 지시문으로 오류 위치가 원래 `.dri` 파일을 가리킵니다.

## CLI 옵션

```
dri <입력.dri> [옵션]
```

| 옵션 | 설명 |
|------|------|
| `--exe <출력파일>` | 실행 파일 생성 |
| `--cpp <출력.cpp>` | C++ 트랜스파일 결과만 저장 |
| `--check` | 컴파일 없이 문법·타입 검사만 수행 |
| `--debug` | 디버그 빌드 (`-O0`, 범위 검사 활성화) |
| `--release` | 릴리스 빌드 (`-O3`, `-march=native`, LTO) |
| `--opt <0~3>` | 최적화 수준 수동 지정 |
| `--native` | `-march=native` 활성화 |
| `--lto` | Link-Time Optimization 활성화 |
| `--trace <파일>` | parallel 루프 추적 파일 생성 (Chrome DevTools 형식) |
| `-D<플래그>` | 컴파일 타임 플래그 정의 (static_if 조건에 사용) |
| `--target <triple>` | 크로스 컴파일 타겟 (`x86_64-linux-gnu`, `aarch64-linux-gnu` 등) |
| `--sysroot <경로>` | 타겟 OS 시스템 헤더/라이브러리 경로 |
| `--cross-cxx <컴파일러>` | 크로스 컴파일러 바이너리 명시 |
| `--source-map <파일>` | JSON 소스맵 생성 (C++ 줄 → dri 줄) |
| `--incremental` | 변경된 파일만 재컴파일 |
| `--no-analyze` | 정적 분석 비활성화 |

## PowerShell / CMD 명령어 차이

dri 명령어 자체는 어느 쉘에서나 동일합니다. 단, **빌드 후 실행**(`&&`) 구문이 쉘마다 다릅니다.

### 컴파일만 (모든 쉘 동일)

```
dri main.dri --exe main.exe
dri main.dri --check
dri main.dri --cpp out.cpp
```

### 컴파일 후 즉시 실행

| 쉘 | 명령어 |
|----|--------|
| **PowerShell 5.1** (기본) | `& "dri.exe" main.dri --exe main.exe; if ($LASTEXITCODE -eq 0) { & ".\main.exe" }` |
| **PowerShell 7+** (pwsh) | `& "dri.exe" main.dri --exe main.exe && & ".\main.exe"` |
| **CMD** | `dri.exe main.dri --exe main.exe && main.exe` |
| **Bash / WSL** | `dri main.dri --exe main && ./main` |

> PowerShell 5.1은 `&&` 연산자가 없습니다 (PowerShell 7에서 추가됨).  
> VSCode 확장의 **dri: Run File** 버튼은 활성 터미널 종류를 자동 감지하여 올바른 문법을 사용합니다.

### 환경변수 확인

| 항목 | PowerShell | CMD |
|------|------------|-----|
| PATH 확인 | `$env:PATH -split ';'` | `echo %PATH%` |
| 설치 경로 확인 | `where.exe dri` | `where dri` |
| dri 버전 확인 | `dri --version` | `dri --version` |

```bash
# 기본 사용
dri main.dri --exe main --release
dri main.dri --cpp out.cpp --check
dri main.dri --exe main --debug
dri main.dri --exe main --opt 2 --native --lto
dri main.dri --exe main --release --trace trace.json
```

| 규칙 | 내용 |
|------|------|
| 문장 종결자 | 모든 문장은 `;`으로 끝납니다 |

| 진입점 | 파일 최상위에 작성 (별도 `main` 함수 불필요) |

```dri

   이렇게 작성합니다.
##

# 한 줄 주석
int x = 42;   # 인라인 주석
print(x);
```

> 변수·타입·입출력 기본 문법은 [[02_기본.md](02_기본.md)]를 참고하세요.

## 크로스 컴파일 (Cross-Compilation)

dri 컴파일러는 `--target` 플래그로 현재 OS와 다른 플랫폼의 바이너리를 빌드할 수 있습니다.

### 지원 타겟

| 타겟 트리플 | 대상 환경 | 설명 |
|------------|---------|------|
| `x86_64-linux-gnu` | Linux x86_64 | HPC 서버, AWS/GCP EC2 |
| `aarch64-linux-gnu` | ARM64 Linux | AWS Graviton, Raspberry Pi 5 |
| `x86_64-w64-mingw32` | Windows 64-bit | Windows 크로스 (Linux→Win) |
| `riscv64-linux-gnu` | RISC-V Linux | 임베디드 Linux 보드 |
| `wasm32-unknown-emscripten` | WebAssembly | 브라우저 실행 |

### 사용 예시

```bash
# 윈도우 개발 PC에서 linux x86_64 AVX-512 바이너리 빌드
dri main.dri --exe main_linux --target x86_64-linux-gnu --release

# ARM64 서버용 빌드 (AWS Graviton)
dri main.dri --exe main_arm64 --target aarch64-linux-gnu --release

# sysroot 지정 (크로스 헤더/라이브러리 경로)
dri main.dri --exe out --target aarch64-linux-gnu --sysroot /opt/aarch64-sysroot

# 명시적 크로스 컴파일러 지정
dri main.dri --exe out --cross-cxx aarch64-linux-gnu-g++
```

### 자동 컴파일러 감지 순서

1. `clang++` 있으면 `--target=<triple>` 플래그로 처리 (가장 유연)
2. 없으면 `<triple>-g++` 바이너리 탐색 (예: `aarch64-linux-gnu-g++`)
3. 둘 다 없으면 오류 + 설치 방법 안내

```bash
# Ubuntu에서 ARM64 크로스 컴파일러 설치
sudo apt-get install gcc-aarch64-linux-gnu g++-aarch64-linux-gnu
```

## 에러 역추적 (Source Mapping)

dri 컴파일러는 생성된 C++ 코드에 `#line` 지시어를 자동 삽입합니다. C++ 컴파일러나 디버거가 에러를 보고할 때 `.dri` 파일의 정확한 줄 번호를 표시합니다.

```bash
# 기본: #line 지시어 자동 삽입 (항상 활성)
dri main.dri --exe main

# 에러 발생 시 출력 예시
main.dri:12:5: error: 'undefined_var' was not declared in this scope
```

### JSON 소스맵 생성

```bash
dri main.dri --cpp out.cpp --source-map out.map.json
```

`out.map.json` 형식:
```json
{
  "version": 3,
  "sourceFile": "main.dri",
  "mappings": [
    {"cppLine": 45, "driFile": "main.dri", "driLine": 12},
    {"cppLine": 52, "driFile": "main.dri", "driLine": 15}
  ]
}
```

### GDB 디버깅

`#line` 지시어 덕분에 GDB도 `.dri` 파일 위치를 추적합니다:

```bash
dri main.dri --exe main --debug --source-map main.map.json
gdb ./main
(gdb) run
# 세그폴트 발생 시: main.dri:23 표시
```
