# 고급 기능
> drv의 고급 언어 기능들: 모듈 시스템, 컴파일 타임 분기, 자동 미분, 빌드 프로파일, 패키지 관리자.

---

## 모듈 시스템 — `module` / `use`

drv는 파일 단위 모듈을 지원합니다. 각 `.drv` 파일은 독립 컴파일 단위이며, `public`으로 선언된 심볼만 외부에 노출됩니다.

### 모듈 선언 — `module`

```drv
# math_utils.drv
module math_utils;

public double fancy_sqrt(double x) {
    return sqrt(x * 2.0);
};

public double lerp(double a, double b, double t) {
    return a + (b - a) * t;
};

private double internal_helper(double x) {
    return x * x;   # 외부에서 접근 불가
};
```

### 모듈 사용 — `use`

```drv
# main.drv
use math_utils;                    # 모듈 전체 임포트
use math_utils::fancy_sqrt;        # 특정 심볼만 임포트

double result = fancy_sqrt(9.0);
print(result);   # 6.0

double v = math_utils::lerp(0.0, 100.0, 0.5);
print(v);        # 50.0
```

### 컴파일러 동작

```
math_utils.drv  →  [독립 컴파일]  →  math_utils.o
main.drv        →  [독립 컴파일]  →  main.o
                →  [링크]         →  실행 파일
```

- 변경된 모듈만 재컴파일 (증분 빌드)
- 심볼 스코프 격리 — 이름 충돌 없음
- 순환 의존 컴파일 타임 감지

## static_if — 컴파일 타임 조건 분기

`static_if`는 **전처리 매크로가 아닙니다.** C++의 `if constexpr`처럼 AST 수준에서 평가됩니다.  
조건이 거짓인 브랜치는 **파싱은 하되 코드 생성을 건너뜁니다** — 텍스트 인라인 방식이 아닙니다.

### 빌드 플래그 조건 (→ C++ `if constexpr` + 빌드 플래그)

```drv
static_if (RELEASE_BUILD) {
    print("release 빌드입니다");
} else {
    print("debug 빌드입니다");
};
```

컴파일 시 `-DRELEASE_BUILD` 플래그가 있으면 then 브랜치만 코드 생성됩니다.

> **모듈 시스템과의 관계**: `static_if`는 텍스트 전처리가 아니므로,  
> `drvpm`으로 배포된 **미리 컴파일된 바이너리 모듈**에도 적용됩니다.  
> 라이브러리가 `static_if`를 사용해도 소스 코드를 전체 공개할 필요가 없습니다.  
> 플래그 값은 링크 타임에 확정됩니다.

### 표현식 조건 (→ C++ `if constexpr`)

```drv
static_if (1 == 1) {
    print("항상 참");
};
```

### `static_if` vs 런타임 `if` 비교

| | `static_if` | `if` |
|--|-------------|------|
| 평가 시점 | 컴파일 타임 | 런타임 |
| 거짓 브랜치 | 코드 생성 안 함 | 코드 생성됨 |
| 조건 종류 | 상수·빌드 플래그 | 모든 표현식 |
| 용도 | 플랫폼/빌드 분기 | 일반 조건 분기 |

## @bench — 실행 시간 측정

함수에 `@bench`를 붙이면 해당 함수의 실행 시간을 자동으로 측정하여 stderr에 출력합니다.

```drv
@bench
void heavy_computation() {
    double sum = 0.0;
    for (int i = 0; i < 1000000; i++) {
        sum = sum + math.sqrt(i as double);
    };
};

heavy_computation();
# stderr 출력: [bench] heavy_computation: 12.345 ms
```

- RAII 방식으로 구현: 함수 진입 시 타이머 시작, 반환 시 자동 측정
- `return`이 여러 개 있어도 정확하게 측정

## @specialize — 타입별 SIMD 특수화

특정 타입에 대해 최적화된 코드 경로를 생성합니다. `@specialize`가 붙은 함수는 `DRV_HOT` 마킹되어 자동 벡터화 우선순위가 높아집니다.

```drv
@specialize
double dot_product(tensor a, tensor b) {
    double result = 0.0;
    for (int i = 0; i < length(a); i++) {
        result = result + a[i] * b[i];
    };
    return result;
};
```

## 자동 미분 (Auto-diff)

drv는 언어 수준에서 자동 미분을 지원합니다.

### diff.forward — 전진 모드 자동 미분

```drv
# f(x) = x^2 + 2x + 1 → f'(x) = 2x + 2
double f(double x) {
    return x * x + 2.0 * x + 1.0;
};

# x = 3.0 에서 f'(3) = 8.0
var df = diff.forward(f, 3.0);
print(df);  # 8.0
```

### diff.numerical — 수치 미분 (중심 차분)

```drv
double g(double x) {
    return math.sin(x);
};

var dg = diff.numerical(g, math.pi() / 4.0);
print(dg);  # cos(π/4) ≈ 0.7071...
```

### diff.hessian — 이계 도함수

```drv
double h(double x) {
    return x * x * x;  # h''(x) = 6x
};

var d2h = diff.hessian(h, 2.0);
print(d2h);  # 12.0
```

## 빌드 프로파일 (Build Profiles)

### Debug 모드

```bash
drv main.drv --exe main.exe --debug
```

- 최적화 수준: `-O0`
- Fast-math 비활성화
- LTO 비활성화
- 런타임 체크 활성화 (배열 범위 검사 등)

### Release 모드

```bash
drv main.drv --exe main.exe --release
```

- 최적화 수준: `-O3`
- `-march=native` (현재 CPU에 맞는 최적화)
- LTO (Link-Time Optimization) 활성화
- Fast-math 활성화

### 수동 설정

```bash
drv main.drv --exe main.exe --opt 2 --native --lto
```

## 패키지 관리자 (drvpm)

별도 실행 파일 `drvpm`으로 제공됩니다. Gradle 방식의 의존성 관리, 증분 빌드, R2 레지스트리 연동을 지원합니다.

### 새 프로젝트 초기화

```bash
drvpm init my-project
cd my-project
```

생성 구조:
```
my-project/
  src/main.drv       # 진입점
  tests/main.drv     # 테스트 스위트
  drvpm.drvi         # 프로젝트 메타데이터
  build/             # 빌드 아티팩트 (자동 생성)
```

### drvpm.drvi 형식 (Gradle 스타일 의존성 관리)

`drvpm.drvi`는 Gradle의 `build.gradle`과 동일한 역할입니다.  
의존성 해석, 플랫폼 조건부 의존성, 개발 전용 의존성을 모두 지원합니다.

```ini
[project]
name        = "my-project"
version     = "0.1.0"
author      = "drewdrew0414"
description = "My drv project"
license     = "MIT"
drv_min     = "0.6.0"           # 최소 drv 컴파일러 버전
registry    = "https://drvpm-registry.cloud"

# ── 런타임 의존성 (배포 포함) ──────────────────────────────────
[dependencies]
math          = "^1.0.0"
tensor        = "~2.3.0"
drv-pandas    = "^0.2.0"
drv-toulmin   = "^0.1.0"
drv-data      = "^0.1.0"
drv-plot      = "^0.1.0"

# ── 개발 전용 의존성 (배포 제외, Gradle devDependencies와 동일) ─
[dev_dependencies]
drv-test      = "^0.1.0"        # 테스트 프레임워크
drv-bench     = "^0.1.0"        # 벤치마크 프레임워크
drv-lint      = "^0.1.0"        # 정적 분석기

# ── 플랫폼 조건부 의존성 (Gradle platform() / configurations와 동일) ─
[dependencies.platform.linux]
cuda-drv      = "^12.0.0"       # Linux에서만 CUDA 연동

[dependencies.platform.windows]
directml-drv  = "^1.0.0"        # Windows에서만 DirectML 연동

[dependencies.platform.macos]
metal-drv     = "^3.0.0"        # macOS에서만 Metal 연동

# ── 피처 플래그 조건부 의존성 ──────────────────────────────────
[dependencies.feature.gpu]
cuda-drv      = "^12.0.0"
opencl-drv    = "^3.0.0"

[dependencies.feature.hpc]
mpi-drv       = "^4.0.0"
blas-drv      = "^3.10.0"

# ── 저장소 설정 (Gradle repositories와 동일) ──────────────────
[repositories]
central   = "https://drvpm-registry.cloud"
local     = "./local_packages"
mirror    = "https://mirror.company.internal/drvpm"

# ── 의존성 해석 전략 ───────────────────────────────────────────
[resolution]
strategy    = "newest-compatible"   # newest-compatible | strict | lowest
fail_on_conflict = true

# ── 태스크 정의 (Gradle tasks와 동일) ──────────────────────────
[tasks]
build   = "drv src/main.drv --exe build/my-project --release"
test    = "drv tests/main.drv --exe build/test && build/test"
lint    = "drv src/main.drv --check"
bench   = "drv bench/main.drv --exe build/bench --release && build/bench"
docs    = "drvdoc src/ --out docs/api/"
deploy  = "drvpm publish"
```

### 의존성 상속 — `extend` (Gradle BOM과 동일)

공통 설정을 부모 프로젝트에서 상속받습니다.

```ini
[project]

extend = "company-base-drvi@^1.0.0"   # 부모 설정 상속

[dependencies]
# 부모의 drv-pandas, drv-data 등 버전 자동 상속
# 오버라이드 필요한 것만 명시
drv-pandas = "^0.3.0"   # 부모 버전 오버라이드
```

### 버전 카탈로그 — `[versions]`

팀 전체 버전을 한 곳에서 관리합니다 (Gradle Version Catalog와 동일).

```ini
[versions]
drv-pandas-ver  = "0.2.3"
drv-plot-ver    = "0.1.5"
arrow-core-ver  = "14.0.0"

[dependencies]
drv-pandas = { version = "$drv-pandas-ver", features = ["arrow", "parquet"] }
drv-plot   = { version = "$drv-plot-ver",   features = ["webgl"] }
```

## @units_check — 물리 단위 검사

`dim` 선언과 함께 사용하여 물리 단위 정합성을 컴파일 타임에 검사합니다.

```drv
dim Meter = "m";
dim Second = "s";
dim Kg = "kg";

Meter distance = Meter(10.0);
Second time = Second(5.0);

# 같은 단위끼리만 덧셈 가능
Meter total = distance + Meter(3.0);  # OK
# Meter wrong = distance + time;       # 컴파일 에러!

# 나누기: 다른 단위 결과
double speed = distance / time;  # m/s (double로 자동 변환)
```

단위 태그가 다른 값은 C++ 타입 시스템이 덧셈/뺄셈을 컴파일 타임에 거부합니다.

> `dim` 물리 단위 타입 선언은 [[02_기본.md](02_기본.md)]를 참고하세요.

## extern "C" — C 라이브러리 연동

HPC 분야의 기존 자산(BLAS, LAPACK, CUDA 등)을 drv에서 직접 호출할 수 있습니다.  
`extern "C"` 블록 안에서 C 함수를 선언하면, drv 컴파일러가 이름 맹글링(Name Mangling) 없이  
C ABI로 링크합니다. 배열 인수는 원시 포인터(`&arr[0]`)로 자동 변환됩니다.

```
extern "C" {
    반환타입 함수명(매개변수);
};
```

```drv
# BLAS dgemm 선언 (행렬 곱셈)
extern "C" {
    void cblas_dgemm(int Order, int TransA, int TransB,
                     int M, int N, int K,
                     double alpha, double[] A, int lda,
                                   double[] B, int ldb,
                     double beta,  double[] C, int ldc);
};

double[] A = new double[1024];
double[] B = new double[1024];
mut double[] C = new double[1024];

# 호출 시 배열은 &A[0] 형태의 원시 포인터로 자동 변환됨
cblas_dgemm(101, 111, 111, 32, 32, 32,
            1.0, A, 32, B, 32,
            0.0, C, 32);
```

여러 함수 선언:

```drv
extern "C" {
    void cblas_daxpy(int n, double alpha, double[] x, int incx,
                     double[] y, int incy);
    double cblas_ddot(int n, double[] x, int incx, double[] y, int incy);
    void cblas_dscal(int n, double alpha, double[] x, int incx);
};
```

| drv 타입 | C 포인터 변환 |
|----------|-------------|
| `double[]` | `double*` (→ `&arr[0]`) |
| `int[]` | `int*` |
| `mut double[]` | `double*` (쓰기 가능) |
| `Borrow<double[]>` | `const double*` |

> **`mut` 키워드**: C 함수가 배열을 수정하는 경우 `mut double[]`로 선언합니다.  
> 선언하지 않으면 컴파일러가 `const` 포인터로 전달해 C 측 쓰기가 UB가 됩니다.

## extern "FFI" — 모던 언어 안전 연동

Rust, Zig 등 **C 호환 ABI + 메타데이터를 제공하는 언어**의 함수와 구조체를 drv 타입 시스템에 안전하게 매핑합니다.  
컴파일러가 타입 정보를 분석해 `Own<T>`, `Ref<T>` 바인딩을 자동 생성합니다.

```
extern "FFI" {
    use 언어::경로::타입;
    fn 함수명(매개변수) -> 반환타입;
};
```

```drv
extern "FFI" {
    # Rust 라이브러리 연동
    use rust::polars::DataFrame;
    use rust::polars::Series;

    fn polars_read_csv(path: String) -> Own<DataFrame>;
    fn polars_filter(df: Ref<DataFrame>, expr: String) -> Own<DataFrame>;
    fn polars_mean(series: Ref<Series>) -> double;
};

Own<DataFrame> df = polars_read_csv("data.csv");
Own<DataFrame> filtered = polars_filter(df, "age > 30");
print(polars_mean(filtered.get_column("salary")));
```

### Own<T> / Ref<T> 자동 매핑 규칙

| 언어 측 | drv 매핑 | 의미 |
|---------|----------|------|
| Rust `Box<T>` / `T` (반환) | `Own<T>` | 소유권 이전 |
| Rust `&T` | `Borrow<T>` | 불변 참조 |
| Rust `&mut T` | `mut Borrow<T>` | 가변 참조 |
| Rust `Arc<T>` | `Ref<T>` | 공유 소유권 |

## @unsafe_legacy extern "C" — 레거시 방어 계층

구형 C, Fortran 라이브러리는 SegFault, 스택 오버플로 등 **프로세스 전체 다운**을 유발할 수 있습니다.  
`@unsafe_legacy`를 붙이면 컴파일러가 각 호출부에 **시그널 핸들러와 격리 래퍼**를 자동 주입합니다.  
라이브러리가 충돌해도 drv 런타임이 신호를 잡아 `Result<T>` 형태로 오류를 반환합니다.

```drv
@unsafe_legacy extern "C" {
    # Fortran LAPACK 루틴 (SegFault 위험 있음)
    void dgesv_(int[] N, int[] NRHS, double[] A, int[] LDA,
                int[] IPIV, double[] B, int[] LDB, int[] INFO);

    # 구형 영상처리 C 라이브러리
    int legacy_encode(mut double[] buf, int size, String format);
};
```

호출 시 컴파일러가 자동 생성하는 래퍼:

```cpp
// 자동 생성 C++ 래퍼 (개념 표시)
Result<void> __drv_safe_dgesv_(...) {
    struct sigaction sa_segv, sa_fpe;
    // SIGSEGV, SIGFPE 핸들러 등록
    // ...
    try {
        dgesv_(...);
        return Ok();
    } catch (...) {
        return Err("legacy call crashed: dgesv_");
    }
}
```

```drv
# drv 코드에서는 Result<>로 안전하게 수신
Result<void> res = dgesv_(n, nrhs, A, lda, ipiv, B, ldb, info);
match res {
    Ok(_)  => { print("LAPACK 성공"); };
    Err(e) => { print("레거시 충돌:", e); };
};
```

### extern 종류 비교

| 구문 | 대상 언어 | 안전성 | 타입 매핑 |
|------|---------|--------|----------|
| `extern "C"` | C (신뢰 가능) | 개발자 책임 | 수동 |
| `extern "FFI"` | Rust, Zig | 컴파일러 자동 검증 | 자동 (`Own`/`Ref`) |
| `@unsafe_legacy extern "C"` | C, Fortran (레거시) | 시그널 격리 자동 주입 | 수동 + 래퍼 |
