# 예제 8: 컴파일러 코드 재배치 최적화

> CPU 파이프라인 스톨, 분기 예측, 루프 언롤링, 메모리 장벽을  
> dri 코드로 직접 관찰하고 최적화하는 방법을 보여 준다.

---

## 개요

현대 CPU는 단순히 "빠른 처리 장치"가 아니라 복잡한 파이프라인 기계다.  
이 예제는 **코드 구조가 하드웨어 성능에 미치는 영향** 4가지를 다룬다.

| 최적화 | 문제 | 해법 |
|--------|------|------|
| 파이프라인 스톨 방지 | `result = result + data[i]` — RAW 해저드 | 4-way 독립 누산기 |
| 분기 예측 | 조건문이 무작위로 참/거짓 → 잘못된 예측 | 데이터 정렬 |
| 루프 언롤링 | `for` 오버헤드 + 자동 벡터화 실패 | `simd for` + step 4 |
| 메모리 장벽 | 컴파일러/CPU가 쓰기 순서를 재배치 | `sys.fence_full()` |

---

## 핵심 개념

### ① 파이프라인 스톨 (RAW 해저드)

```dri
# 비효율: 매 반복이 직전 결과를 기다림 → 스톨
double sum_serial(double[] data) {
    double result = 0.0;
    for (i in 0..lst.length(data)) {
        result = result + data[i];
    }
    return result;
}

# 효율: 4개 독립 누산기 → CPU 슈퍼스칼라 병렬 실행
@bench
double sum_4accum(double[] data) {
    double s0 = 0.0; double s1 = 0.0;
    double s2 = 0.0; double s3 = 0.0;
    # ... (step 4 루프로 동시 갱신)
    return s0 + s1 + s2 + s3;
}
```

**원리**: `s0·s1·s2·s3` 는 서로 의존하지 않으므로  
CPU Out-of-Order 엔진이 4개를 동시 실행 → 약 3-4× 처리량 향상.

### ② 분기 예측

```dri
# 정렬 전: 양수/음수 무작위 → 분기 예측 실패율 ~50%
int count_positive_unsorted(double[] data) { ... }

# 정렬 후: 양수가 앞쪽에 몰림 → 예측 성공률 ~99%
int count_positive_sorted(double[] data) {
    double[] sorted = lst.sort(data);
    ...
}
```

### ③ 루프 언롤링 + SIMD

```dri
# 기본: 매 반복마다 분기 체크 + 인덱스 증가
void zero_normal(mut Borrow<double[]> arr) {
    for (i in 0..lst.length(arr)) { arr[i] = 0.0; }
}

# SIMD: 자동 벡터화 힌트 → 한 번에 4~8원소 처리
void zero_simd(mut Borrow<double[]> arr) {
    simd for (i in 0..lst.length(arr)) { arr[i] = 0.0; }
}

# 수동 언롤: step 4로 루프 오버헤드 제거
void zero_unrolled(mut Borrow<double[]> arr) {
    int n = lst.length(arr);
    int n4 = n - (n % 4);
    for (i in 0..n4 step 4) {
        arr[i] = 0.0; arr[i+1] = 0.0; arr[i+2] = 0.0; arr[i+3] = 0.0;
    }
}
```

### ④ 메모리 장벽

```dri
# sys.fence_full() — 모든 읽기/쓰기가 이 줄 이전에 완료됨을 보장
void safe_producer(mut Borrow<int[]> shared, int val) {
    shared[0] = val;
    sys.fence_full();    # 소비자가 data보다 먼저 ready를 보지 못하게 방지
    shared[1] = 1;       # ready flag
}
```

## 핵심 문법 정리

| 구문 | 의미 |
|------|------|
| `@bench` | 함수 실행 시간을 stderr에 출력 |
| `simd for (i in ...)` | OpenMP `simd` 벡터화 힌트 |
| `for (i in 0..n step 4)` | `i += 4` 증가 루프 (수동 언롤링) |
| `sys.fence_full()` | `std::atomic_thread_fence(seq_cst)` |
| `sys.fence_acquire()` | `std::atomic_thread_fence(acquire)` |
| `mut Borrow<T[]>` | `std::vector<T>&` — 원본 수정 가능 |
