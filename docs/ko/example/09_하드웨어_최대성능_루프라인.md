# 예제 9: 하드웨어 최대 성능 — Roofline 모델

> STREAM 벤치마크, 캐시 계층 대역폭 측정, Roofline 모델 시각화를  
> dri 코드로 구현하여 하드웨어 한계를 직접 관측한다.

---

## 개요

어떤 알고리즘이든 두 가지 하드웨어 한계 중 하나에 부딪힌다.

```
성능 [GFLOP/s] = min(Peak_FLOPS,  Arithmetic_Intensity × Peak_BW)
                        ↑               ↑                   ↑
                   계산 한계       산술 강도           메모리 한계
```

**Roofline 모델**은 이 관계를 시각화해 병목이 계산인지 메모리인지 판단한다.

| 용어 | 정의 |
|------|------|
| Arithmetic Intensity (AI) | FLOP ÷ 접근한 바이트 수 |
| Memory-Bound | AI < Ridge Point → 메모리 대역폭이 병목 |
| Compute-Bound | AI > Ridge Point → 계산 처리량이 병목 |
| Ridge Point | Peak_FLOPS ÷ Peak_BW [FLOP/Byte] |

---

## 핵심 개념

### STREAM 벤치마크

```dri
@bench
void stream_triad(Borrow<double[]> a, Borrow<double[]> b,
                  mut Borrow<double[]> c, double scalar) {
    for (i in 0..lst.length(c)) {
        c[i] = a[i] + scalar * b[i];
    }
}
```

STREAM Triad는 메모리 대역폭의 표준 지표다.  
`3N × 8 bytes`를 읽/쓰므로 `AI = 1/12 FLOP/Byte`.

### 캐시 계층 스윕

```dri
# 작은 배열 (L1 캐시 안): ~수백 GB/s
# 중간 배열 (L2/L3):       ~수십 GB/s
# 큰 배열  (RAM):           ~10-20 GB/s
for (size_mb in [0, 1, 8, 64]) {
    # 반복 측정으로 캐시 효과 포착
}
```

### Roofline 텍스트 시각화

```
▲ GFLOP/s
│    ╔══════════════════ Peak FLOPS = XXX GFLOP/s
│    ╟────────────────────
│   /  compute-bound 영역
│  /
│ / ← 기울기 = Peak BW
│/___________________________▶ AI [FLOP/Byte]
   Ridge Point
```

## 핵심 문법 정리

| 구문 | 의미 |
|------|------|
| `Borrow<double[]>` | `const std::vector<double>&` |
| `mut Borrow<double[]>` | `std::vector<double>&` |
| `@bench` | 함수 단위 실행 시간 측정 |
| `parallel for` | OpenMP 병렬 루프 |
| `sys.time_ms()` | 현재 시각 (밀리초, 고정밀) |
| `lst.fill(N, 0.0)` | 크기 N, 초깃값 0.0인 리스트 생성 |
